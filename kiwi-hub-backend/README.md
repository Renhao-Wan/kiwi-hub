# KiwiHub (奇异果社区) - 项目设计文档

## 🚀 部署指南

本项目针对低配服务器（2核2G）进行了深度优化，移除了 Gateway 改用 Nginx，并使用云端中间件。

👉 **[点击查看Kiwi-Hub 容器化部署指南 (基于 Nginx 分支)](docs/deployment.md)**

## 1. 项目背景与简介

**KiwiHub** 是一个基于微服务架构构建的**高性能社交内容分享平台**。该项目旨在为用户提供一个轻量、实时、高互动的社区环境。系统采用 **混合存储架构（MySQL + MongoDB）** 与 **Redis 缓存**，实现了从用户注册、内容发布、多级评论互动到短链接分享的全链路业务，兼顾结构化数据的可靠性与非结构化数据的灵活性。

关于命名：为什么叫 "KiwiHub"？

- **Kiwi (奇异果):** 象征着**新鲜**与**活力**。寓意社区内容的实时性（Freshness）和高营养价值（Quality），给人一种年轻、极客的清爽感。
- **Hub (枢纽):** 代表连接。这里不仅是内容的存储地，更是用户与用户、用户与技术之间连接的中心。



## 2. 技术架构与选型

本项目采用 **Spring Cloud Alibaba** 微服务生态体系，数据存储采用 **混合存储架构（MySQL + MongoDB）** 与 **Redis 缓存**，兼顾结构化数据的可靠性与非结构化数据的灵活性。

| **类别**     | **技术栈**                           | **选型理由**                                                 |
| ------------ |-----------------------------------| ------------------------------------------------------------ |
| **核心框架** | Spring Boot 3.4.12 + Spring Cloud Alibaba | 工业界主流微服务解决方案，生态完善。                         |
| **服务治理** | Nacos + Nginx                     | 实现服务注册发现、配置管理。                   |
| **核心存储** | **MySQL + MongoDB**               | **MySQL** 存储结构化元数据（文章统计、评论、点赞、关注关系）；**MongoDB** 存储非结构化内容（文章正文、用户画像），支持海量数据水平扩展。 |
| **高速缓存** | **Redis**                         | 用于短链接映射、点赞计数、热点数据缓存、分布式锁及布隆过滤器。 |
| **对象存储** | **七牛云 OSS**                       | 图片、文章、视频等静态资源的云端托管。                       |



## 3. 微服务模块拆解与职责

系统被拆分为三个核心业务微服务，通过 RESTful API 进行交互。

### 1. 用户中心服务 (kiwi-user)

**定位：** 混合存储的用户中心服务。

- **核心职责：**
  - **身份认证：** 负责用户的注册、登录，使用分布式Session。
  - **画像管理：** 利用 MongoDB 灵活的文档结构，存储用户多变的属性（头像、签名、标签、社交链接），无需频繁变更表结构。
  - **关注体系：** MySQL 存储关注与粉丝关系表（Follow/Fans），支持快速查询"我关注的人"和"粉丝列表".



###  2. 内容交互服务 (kiwi-content)

**定位：** 混合存储的内容交互服务。

- **核心职责：**
  - **文章管理：** 文章的发布、编辑与删除。**MySQL** 存储文章元数据（标题、摘要、标签、统计信息）；**MongoDB** 存储文章正文（支持富文本、Markdown、长文本），图片/视频上传至七牛云 OSS，数据库仅存 URL。
  - **盖楼评论：** 设计嵌套式（Nested）或树形结构的评论系统。**MySQL** 存储评论元数据（层级结构、时间顺序）；利用 MongoDB 单文档原子性支持高并发下的评论写入。
  - **点赞聚合：** 使用 **Redis Set/HyperLogLog** 实现高性能点赞与去重。**MySQL** 存储点赞关系（userId + articleId 组合主键）；通过 **@Async** 异步同步点赞数到 MongoDB，避免高频写库.



###  3. 短链接服务 (kiwi-link)

**定位：** 高并发的基础设施工具。

- **核心职责：**
  - **长短链转换：** 将站内的长文章链接压缩为短链接（如 `kiwi.hub/x7z9A`），便于在第三方平台分享。
  - **算法实现：** 采用 **MurmurHash + Base62** 编码生成短码。
  - **防重与缓存：** 引入 **布隆过滤器 (Bloom Filter)** 快速判断短码是否已存在，防止哈希冲突；利用 Redis 缓存热点映射关系，提升重定向速度.



### 4. 项目亮点

1. **混合存储架构实践：** 采用 **MySQL + MongoDB + Redis** 三层存储策略，MySQL 保证结构化数据的可靠性和事务一致性，MongoDB 处理海量非结构化内容，Redis 缓存热点数据，构建了高可用的多级存储体系。
2. **高性能短链系统：** 独立拆分短链接服务，解决了海量短码生成的冲突检测与高并发读取问题，具备工业级的系统设计思维。
3. **读写分离与冷热分离：** 静态资源上云（OSS），热点数据入缓存（Redis），持久化数据落库（MySQL/Mongo），构建了多级存储体系。
